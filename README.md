# 弱男モンスター（弱モン）

> 地下アイドルが特典会でヲタクに色恋営業を仕掛け、集めたお金と手持ちのヲタクでライバルアイドルとバトルする HTML5 ブラウザゲーム

## ゲーム概要

「弱男モンスター」（略称：弱モン）は、地下アイドルのリアルな生態系をモチーフにした **ポケモン風ターン制バトルゲーム** です。プレイヤーは地下アイドルとなり、特典会でヲタク（弱男＝弱モン）を色恋営業で手懐け、集めたヲタクの財力を使ってライバルアイドルとの頂上決戦に挑みます。

## ゲームフロー

```
タイトル → アイドル選択 → 特典会（色恋営業）→ バトル → 次のステージ or リトライ
```

### 1. アイドル選択

5人のプレイアブルアイドルから1人を選択します。各アイドルは固有スキルと特性ボーナスを持ちます。

| アイドル | 特性 | ヘアスタイル |
|---------|------|------------|
| りん | ガチ恋製造機（好感度UP） | ツインテール |
| あいか | スパチャの女王（金額UP） | ロングストレート |
| そら | 天然キラー（捕獲率UP） | ポニーテール |
| れいな | 炎上系アイドル（攻撃力UP） | ショートボブ |
| ゆめ | 病みカワ（全ステータスUP） | アシンメトリー |

### 2. 特典会フェーズ（色恋営業）

ステージごとに訪れるヲタクに対して色恋営業アクションを実行し、**好感度を上げて捕獲** ＆ **お金を稼ぎます**。

| アクション | 好感度 | 獲得金額 | 効果 |
|-----------|-------|---------|------|
| 手を握る | +10 | ¥1,000 | ヲタクが赤面 |
| 目を見つめる | +20 | ¥3,000 | キラキラエフェクト |
| 2ショットチェキ | +30 | ¥5,000 | カメラフラッシュ |
| 「好きだよ♡」と囁く | +50 | ¥10,000 | 目がハートに |
| ハグする | +80 | ¥30,000 | ヲタクが昇天 |
| 指チュー | +100 | ¥50,000 | ヲタクが完全に堕ちる |

### 3. バトルフェーズ

捕獲したヲタクを使い、ライバルアイドルのヲタクとポケモン風ターン制バトルで戦います。

### 4. ステージ構成

全5ステージを勝ち抜いて東京ドームを制覇しましょう。

1. **小さなライブハウス** — 駆け出しアイドル
2. **地下アイドルの聖地** — 中堅アイドル
3. **大型特典会イベント** — 人気アイドル
4. **武道館** — トップアイドル
5. **東京ドーム** — 伝説のアイドル

## 技術スタック

### ゲームエンジン

- **[Phaser 3](https://phaser.io/)** (v3.90+) — HTML5 ゲームフレームワーク。Scale Manager による画面自動フィット（`Phaser.Scale.FIT`）、シーン管理、Tween アニメーション、パーティクルエフェクト等を使用

### 言語・ビルドツール

- **TypeScript** (v6) — `strict: true` で型安全を確保。ESModules (`module: ESNext`)
- **Vite** (v8) — 開発サーバー（HMR）とプロダクションビルド。内部的に Rolldown を使用
- **Node.js** (v20+ / LTS)

### グラフィック

- **Canvas 2D API** — 全キャラクター・UI テクスチャを **100% プログラム描画**。外部画像アセットは一切使用せず、`CanvasRenderingContext2D` の `ellipse`, `bezierCurveTo`, `createRadialGradient` 等で描画
  - アイドル: パワーパフガールズ風の巨大な丸目 + アイカツ風フリル衣装
  - ヲタク: タイプ別 5 種 × 表情 5 パターン（通常/赤面/キラキラ/ハート目/昇天）= 25 テクスチャ

### フォント

- **[M PLUS Rounded 1c](https://fonts.google.com/specimen/M+PLUS+Rounded+1c)** — Google Fonts。ゲーム全体のUI テキストに使用

### CI/CD・ホスティング

- **GitHub Actions** — `main` ブランチへの push で自動ビルド＆デプロイ
- **GitHub Pages** — 静的サイトとしてホスティング

```
jakudanmonster/
├── index.html                # エントリーポイント (1280×720, レスポンシブ)
├── package.json
├── tsconfig.json              # strict: true, ES2020 target
├── vite.config.ts             # base: '/jakudanmonster/'
├── .github/workflows/
│   └── deploy.yml             # GitHub Actions → GitHub Pages
└── src/
    ├── main.ts                # Phaser.Game 初期化 (Scale.FIT, 6シーン)
    ├── utils/
    │   ├── constants.ts       # 型定義, タイプ相性, アクション, 技, ステージ
    │   ├── SpriteGenerator.ts # Canvas 2D 描画ユーティリティ
    │   └── SaveManager.ts     # localStorage セーブ/ロード
    ├── characters/
    │   ├── IdolData.ts        # 5人のアイドル定義
    │   ├── OtakuData.ts       # ヲタク生成ロジック
    │   ├── IdolGenerator.ts   # アイドル Canvas 描画 (パワパフ顔+アイカツ衣装)
    │   └── OtakuGenerator.ts  # ヲタク Canvas 描画 (5タイプ×5表情)
    ├── battle/
    │   ├── BattleSystem.ts    # ターン制バトルエンジン (タイプ相性計算)
    │   └── MoneyManager.ts    # 所持金管理
    └── scenes/
        ├── BootScene.ts       # テクスチャ一括生成
        ├── TitleScene.ts      # タイトル画面 (パーティクル演出)
        ├── IdolSelectScene.ts # アイドル選択 UI
        ├── TokutenkaiScene.ts # 特典会フェーズ (6アクション+エフェクト)
        ├── BattleScene.ts     # バトル UI (HP バー, 技選択, 勝利演出)
        └── ResultScene.ts     # リザルト画面
```

## 開発

```bash
npm install        # 依存パッケージインストール
npm run dev        # 開発サーバー起動 (http://localhost:5173/jakudanmonster/)
npm run build      # プロダクションビルド (dist/)
npm run preview    # ビルド結果のプレビュー
```

## デプロイ

`main` ブランチへの push で GitHub Actions により自動ビルド＆GitHub Pages にデプロイされます。
